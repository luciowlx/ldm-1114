# CONSENSUS — 任务管理简化改造

更新时间：2025-10-31 00:20（UTC+8）

## 1. 明确的需求描述
- 目标：移除 TaskManagement 创建/编辑流程中的 JSON 配置模式，仅保留页面配置对象模式。
- 范围：前端原型（React+TS），不涉及真实后端；统一生成对象结构的 `task.config`（含 output 配置），支持编辑旧任务的对象配置回显。

## 2. 技术实现方案与约束
- 类型与状态：移除 `hyperparameterMode`、`manualConfig` 等字段；`FormData` 仅保留页面配置对象相关字段。
- UI：删除模式切换 `RadioGroup` 与 JSON 文本域及导入/导出按钮；参数配置页直接展示页面配置。
- 逻辑：创建/编辑时统一构建对象模式的 `config`；时序预测兼容生成 `mainVariableFile = mainVariableFiles[0]`。
- 验证：`validateForm` 仅校验页面配置参数；不再对 JSON 文本进行解析或校验。
- 约束：保持与现有架构一致；复用已有组件与类型；函数级注释补充。

## 3. 任务边界与验收标准
- 边界：仅影响 TaskManagement.tsx；其他模块不强制改造（若引用冲突再评估）。
- 验收标准：
  1) 参数配置页无“配置模式”切换与 JSON 面板；
  2) 创建与编辑任务均生成对象结构的 `config`，含 `output`；
  3) 类型与校验无 JSON 相关逻辑；
  4) DevServer 运行无编译错误与控制台错误；
  5) 说明文档与对齐文档同步更新。

## 4. 不确定性处理
- 若后续需要恢复“导入模板”能力，将基于页面配置表单的预填功能做拓展；当前版本不提供。
